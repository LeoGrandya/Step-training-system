# 项目压缩前须知

在打包、压缩或迁移本仓库前，可先清理**运行时数据**以减小体积；下列路径均为「可删除的数据」类内容（删除后果已注明）。**请勿删除源码与配置文件**。

路径以仓库根目录 `pose3d_project_3.0/` 为基准。

**打包最小可运行集**时可排除（不影响 `python web_1/v1.py` + Vue 构建）：`_archive/`、`node_modules/`、各子目录 `.venv/`、`video_sync/.venv/`。

---

## 1. 主应用 `web_1`（最推荐删除以下内容）


| 路径                             | 说明                                                                                                                               |
| ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------- |
| `web_1/jobs/`                  | 每次视频分析任务目录（上传视频、同步/pose3d 中间产物、kinematics CSV、报告 JSON、流水线日志等）。已在 `web_1/.gitignore` 中忽略。整目录删除可显著减小体积；**已提交的任务结果将丢失**（仅磁盘上的任务产物）。 |
| `web_1/data/app.db`            | SQLite 数据库：**用户、历史报告等**业务数据。删除 = **清空本地业务库**。若存在 `app.db-shm`、`app.db-wal` 可一并删除（需先停止占用数据库的进程）。                                  |
| `web_1/data/profiles.jsonl`    | `POST /api/profile` 追加的配置上报日志。删除仅丢失历史记录，不影响程序运行。                                                                                 |
| `web_1/data/analysis_results/` | 历史分析结果索引（如 `items/res_*.json`）。删除后「历史结果」列表可能变空；与 `jobs/` 中报告的关系视使用情况而定。                                                          |
| `web_1/artifacts/immutable/`   | 阶段产物缓存（`ArtifactCache`）。删除后下次分析可能重新生成，**体积可降、首次计算可能变慢**。                                                                         |


---

## 2. 全仓库通用（缓存与环境）


| 路径                         | 说明                                                                            |
| -------------------------- | ----------------------------------------------------------------------------- |
| 任意目录下的 `__pycache__/`      | Python 字节码缓存，可删；运行后会再生成。                                                      |
| 任意 `*.pyc`、`*.pyo`         | 编译缓存文件，可删。                                                                    |
| `.venv/`、`venv/`（根目录或子项目内） | Python 虚拟环境；**删后需重新** `python -m venv` 并 `pip install -r ...`。压缩「纯源码包」时常排除此项。 |


---

## 3. 其它子模块与打包产物


| 路径                                                                                        | 说明                                                    |
| ----------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| `dist/`                                                                                   | `web_1/.gitignore` 中提及的发布归档目录（若存在），可删。                |
| `video_sync/`、`pose3d_project_2.0_fixed/`、`footwork_kinematics_universal2/` 等子目录内的输出、临时文件 | 若曾在这些目录运行脚本，可能产生中间结果；**以各子目录 README 与实际文件为准**，按需手动清理。 |


---

## 4. 不建议删除（非「数据清理」范畴）

- 全部 `.py`、`.html`、`.js`、`.css`、模板与配置、标定示例等**源码与静态资源**。
- 除非明确要「重置系统」：否则勿随意删除 `**web_1/data/app.db`**，以免丢失用户与报告记录。

---

## 5. 常见清理策略（按目标）

- **尽量减小压缩包、保留用户与报告**：优先删 `web_1/jobs/`、可选删 `web_1/artifacts/immutable/`、删各 `__pycache__/`；保留 `web_1/data/app.db` 与 `web_1/data/analysis_results/`（若需保留历史索引）。
- **接近纯源码分发**：排除 `.venv/`、`web_1/jobs/`、`web_1/data/*.db`*、`web_1/data/profiles.jsonl`、`web_1/data/analysis_results/`、`web_1/artifacts/`，并排除所有 `__pycache__/`。

---

## 6. 参考配置

- 路径定义见：`web_1/v1.py` 中 `DATA_DIR`、`JOBS_ROOT`、`ARTIFACT_CACHE_ROOT`、`RESULT_STORE` 等。
