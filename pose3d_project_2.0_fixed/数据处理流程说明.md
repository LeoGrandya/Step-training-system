# 人体姿态数据处理流程说明

本文档说明从双目视频输入到最终三维姿态数据输出的完整处理流程。所有步骤集成在统一项目中，一次运行即可走通全链路。

---

## 完整流程概览

```text
双目视频 (left/right)
    │
    ▼
相机标定 (stereo_params.json / 地面控制点标定)
    │
    ▼
MediaPipe 2D 姿态检测（33 点 + 2 脚方向点 = 35 点）
    │
    ▼
双目三角化 → 3D 绝对坐标 + 虚拟关键点(body_com)
    │
    ▼
地面 Z 轴对齐 → 导出宽表 (pose3d_wide.csv)
    │
    ▼
第一步：异常值处理（滑动窗口 MAD）
    │
    ▼
第二步：短缺失插值（PCHIP）
    │
    ▼
第三步：低通滤波（四阶巴特沃斯）
    │
    ▼
最终输出 (pose3d_wide_processed.csv)
```

---

## 视频分析与 3D 重建

### 1.1 输入：双目视频

输入为一对同步的双目视频（left/right），文件结构如下：

```text
data/<group>/<person>/
├─ left/
│  └─ video.mp4
├─ right/
│  └─ video.mp4
└─ stereo_params.json          # 双目相机标定参数
```

支持的视频格式：`.mp4`、`.avi`、`.mov`、`.mkv`。

### 1.2 相机标定与地面控制

#### 双目相机标定

每个 group 目录下需提供 `stereo_params.json`，包含左右相机的内参矩阵（K1/K2）、畸变系数（dist1/dist2）、旋转矩阵（R）、平移向量（T）和图像尺寸。这些参数用于构建投影矩阵，将 2D 像素坐标三角化为 3D 空间坐标。

#### 地面控制点标定

采用**地面单应性射线法（ground homography rays）**进行三维重建。在每组视频的第 0 帧，手动标注四个地面控制点（TL → TR → BR → BL），构建图像平面到世界平面（z=0）的单应性矩阵。该方法的优势在于不依赖严格的立体匹配，而是通过射线交汇确定 3D 位置，对遮挡和匹配误差具有一定鲁棒性。

标定信息保存在 `camera_pose_summary.json` 中，包含左右相机的投影矩阵、单应性矩阵和相机光心世界坐标。

### 1.3 MediaPipe 2D 姿态检测

采用 **MediaPipe Pose Landmarker**（full 模型）对左右视频逐帧进行 2D 人体关键点检测。

**检测参数：**

| 参数 | 默认值 | 说明 |
|---|---|---|
| 模型 | pose_landmarker_full.task | 全量姿态关键点模型 |
| `min_pose_detection_confidence` | 0.4 | 姿态检测置信度阈值 |
| `min_pose_presence_confidence` | 0.4 | 姿态存在置信度阈值 |
| `min_tracking_confidence` | 0.4 | 追踪置信度阈值 |
| `num_poses` | 1 | 单人检测模式 |
| 输出格式 | mediapipe35 | 33 个官方关键点 + 2 个脚方向点 |

输出 35 个关键点（mediapipe35 格式），包含标准的 33 个 MediaPipe 人体关键点，额外增加左右脚方向点（left_foot_direction / right_foot_direction）。

### 1.4 双目三角化

利用左右相机的投影矩阵，将成对的 2D 关键点三角化为 3D 空间坐标。

**三角化条件：**
- 左右相机中该关键点均被检测到（`detected = 1`）
- 可见性（visibility）≥ 0.5
- 存在性（presence）≥ 0.5
- 重投影误差不超过设定阈值（可选）
- 地面单应性射线法下，左右射线间隙 ≤ 0.30 m

满足条件的点标记为有效（`valid = 1`），不满足的点坐标为空。

**虚拟关键点计算：**
- **body_com（近似重心）**：由髋关节中点（55%）、肩关节中点（30%）、脚部中点（15%）加权合成
- **left_foot_direction / right_foot_direction（脚方向点）**：由脚跟（heel）指向脚趾（foot_index），按 0.60 倍延伸

### 1.5 坐标系统与地面对齐

**坐标轴定义（绝对坐标系）：**
- x 轴：地面横向
- y 轴：地面纵向
- z 轴：竖直向上（正方向）

重建后的 3D 坐标经过 z 轴翻转（`axis_signs = (1, 1, -1)`）以满足上述定义。

**地面 Z 轴对齐：**选取双脚跟（heel）、脚踝（ankle）、脚趾（foot_index）共 6 个关键点的 z 坐标均值作为地面参考高度，整体平移使该参考高度对齐至 z = 0。

### 1.6 宽表导出

将长表格式（每行 = frame_id + joint_id + joint_name + x/y/z）转换为宽表格式（每行 = frame_id + body_com_x/y/z + ... + right_foot_index_x/y/z），输出为 `pose3d_wide.csv`。

宽表包含 11 个关键点（按列序）：body_com、left_hip、right_hip、left_knee、right_knee、left_ankle、right_ankle、left_heel、right_heel、left_foot_index、right_foot_index，每个点含 x/y/z 三列，共 33 列坐标数据。

## 数据后处理

默认采样率：**100 Hz**。以下各步骤均基于该帧率设定默认参数。

### 2.1 第一步：异常值处理

#### 目的

三维姿态数据在采集过程中可能因遮挡、反光、标注误差等原因产生明显偏离真实运动轨迹的异常点。这些异常点若不处理，会严重干扰后续的运动学计算（如速度、加速度）。本步骤的目标是自动检测并标记这些异常值，将其替换为 `NaN`，使其不参与后续数值运算。

#### 算法

采用**滑动窗口中位数结合绝对中位差（Median Absolute Deviation, MAD）**的方法进行逐帧逐轴检测。对于每一帧，在以该帧为中心的局部窗口内计算中位数和 MAD，若该帧的观测值偏离局部中位数超过给定阈值，则判定为异常。

MAD 的定义为：

$$\text{MAD} = \text{median}(|X_i - \text{median}(X)|)$$

其中 $X$ 为滑动窗口内的所有有效观测值。

#### 参数设置

| 参数 | 默认值 | 说明 |
|---|---|---|
| `window_size` | 19 | 滑动窗口大小（帧），必须为奇数。100 Hz 下 19 帧覆盖约 0.19 秒 |
| `mad_k` | 3.5 | 异常判定阈值倍数。$k=3.5$ 在快速变向动作中能有效减少误判 |
| `use_scaled_mad` | True | 是否使用缩放因子 $1.4826$ 将 MAD 近似为标准差尺度 |
| `min_mad` | $10^{-6}$ | MAD 下限，防止除零或过小的 MAD 导致正常点被误判 |
| `replace_with_nan` | True | 检测到的异常值替换为 `NaN`，确保不参与后续运动学计算 |

#### 缩放因子说明

对于正态分布数据，MAD 与标准差 $\sigma$ 之间存在关系：

$$\sigma \approx 1.4826 \times \text{MAD}$$

在计算阈值时，将 MAD 乘以该缩放因子后再乘以 $k$，使阈值近似为 $k \times \sigma$。对于非正态数据（如人体运动数据），该近似仍提供合理的异常判定基准。

#### 判定规则

对于坐标序列 $x_t$（$t$ 为帧索引），在以 $t$ 为中心的大小为 $w$ 的窗口内：

1. 计算局部中位数 $\tilde{x}_t = \text{median}(x_{t-\lfloor w/2 \rfloor}, \dots, x_{t+\lfloor w/2 \rfloor})$
2. 计算局部 MAD：$\text{MAD}_t = \text{median}(|x_i - \tilde{x}_t|)$
3. 计算缩放后的阈值：$T_t = k \times 1.4826 \times \max(\text{MAD}_t, \text{min\_mad})$
4. 若 $|x_t - \tilde{x}_t| > T_t$，则 $x_t$ 被标记为异常并替换为 `NaN`

---

### 2.2 第二步：短缺失插值

#### 目的

经过异常值处理后，数据中可能存在两类缺失：原始数据本身的缺失，以及上一步被标记为异常并替换为 `NaN` 的位置。短时间的缺失可以通过插值合理恢复，避免因少量缺失导致整段数据不可用。

#### 算法

采用 **PCHIP（Piecewise Cubic Hermite Interpolating Polynomial，分段三次埃尔米特插值）**方法。与三次样条插值（Cubic Spline）不同，PCHIP 在保证一阶导数连续的同时，通过限制导数值来**保持曲线的单调性**，避免在数据突变处产生过冲（overshoot）或振荡。这一"保形"特性使其特别适合人体运动数据——运动轨迹通常包含急停、变向等非平滑特征，PCHIP 不会在这些位置产生虚假的"回弹"。

#### 插值范围限制

只对满足以下**全部条件**的缺失段进行插值：

1. **缺失长度** ≤ `max_gap_frames` 帧（默认 25 帧，即 100 Hz 下 ≤ 0.25 秒）
2. **不在序列两端**：缺失段不在序列的开头或结尾（避免外推）
3. **前后有有效值**：缺失段的前一帧和后一帧均有有效数据

超过最大长度的缺失段保持为 `NaN`，不强行插值。

#### 参数设置

| 参数 | 默认值 | 说明 |
|---|---|---|
| `method` | pchip | 插值方法。`pchip` 保形无过冲；`makima`/`akima` 曲线更自然但可能有过冲；`linear` 为线性兜底 |
| `max_gap_frames` | 25 | 最大插值长度（帧）。100 Hz 下 25 帧 = 0.25 秒 |
| `min_valid_points` | 4 | 三次插值所需最少有效点数。不足时自动退化为线性插值 |
| `inside_only` | True | 仅插值内部缺失，不处理首尾 |

---

### 2.3 第三步：低通滤波

#### 目的

经过插值后的数据虽已无异常值，但仍可能包含因采集噪声、标注抖动等引入的高频分量。这些高频噪声在计算速度和加速度时会被差分运算放大，严重影响运动学分析的可靠性。本步骤使用低通滤波平滑信号，抑制高频噪声。

#### 算法

采用**四阶巴特沃斯（Butterworth）低通滤波器**，并使用**零相位滤波（forward-backward filtering）**消除相位延迟。

巴特沃斯滤波器的特点是在通带内频率响应最为平坦，通带到阻带的过渡平滑且单调递减。四阶设计在阻带提供约 24 dB/倍频程的衰减速率，能有效抑制高频噪声。

零相位滤波通过正向-反向两次滤波实现：第一次滤波后，将信号反转再滤波一次，最后再反转回来。这样正向滤波引入的相位延迟被反向滤波抵消，滤波后的信号在时间轴上不会发生偏移，保证运动事件（如峰值速度所在的帧）不被位移。

#### 参数设置

| 参数 | 默认值 | 说明 |
|---|---|---|
| `method` | butterworth | 滤波方法。`butterworth` 为巴特沃斯零相位低通；`savgol` 为 Savitzky-Golay 平滑 |
| `butterworth_order` | 4 | 巴特沃斯滤波器阶数。4 阶在平滑度和响应速度间取得平衡 |
| `cutoff_hz` | 6.0 | 截止频率（Hz）。6 Hz 适合快速步伐动作，可根据实际反馈调整为 7-8 Hz |
| `savgol_window` | 19 | Savitzky-Golay 窗口大小（帧），必须为奇数。备选方案 |

#### 截止频率选择依据

人体运动的主要信号能量通常集中在较低频段。以步伐运动为例：

- 步行：主要频率成分在 0.5–3 Hz
- 跑步/快速运动：主要频率成分在 1–6 Hz
- 高频噪声（传感器噪声、标注抖动）：通常在 10 Hz 以上

选择 6 Hz 作为默认截止频率可在保留运动主频成分的同时有效抑制高频噪声。

#### Savitzky-Golay 备选方案

Savitzky-Golay 滤波器通过在滑动窗口内拟合多项式实现平滑。其优势在于更好地保留信号的局部峰值和形状，但高频衰减特性不如巴特沃斯滤波器可控。当巴特沃斯滤波过度平滑运动细节时，可尝试切换为 Savitzky-Golay。

---

## 输出

整个流程的最终产物为每个 pair 输出一份处理后的 CSV 文件（`pose3d_wide_processed.csv`），包含经过异常值处理、短缺失插值和低通滤波后的全部关节点三维坐标。

输出文件结构与输入宽表一致：`frame_id` + 各关节点 `_x` / `_y` / `_z` 列。

## 参数调节指南

### 视频分析与 3D 重建参数（`pose3d_pkg/user_params.py`）

| 现象 | 调节方向 |
|---|---|
| 3D 重建后某关节频繁缺失 | 降低 `min_visibility` / `min_presence`（如 0.5 → 0.3） |
| 3D 重建出现明显错位点 | 降低 `max_ray_gap_m`（如 0.30 → 0.15），收紧射线间隙限制 |
| 地面高度整体偏移 | 检查地面控制点标定，或调整 `ground_alignment.target_z` |

### 异常值检测（`user_params.py` postprocess.outlier）

| 现象 | 调节方向 |
|---|---|
| 正常急停/变向被误判为异常 | 增大 `mad_k`（如 3.5 → 4.0），或增大 `window_size`（如 19 → 25） |
| 明显异常点未被检测到 | 减小 `mad_k`（如 3.5 → 2.5），或减小 `window_size`（如 19 → 11） |

### 插值（`user_params.py` postprocess.interpolation）

| 现象 | 调节方向 |
|---|---|
| 插值后曲线在缺失段出现不自然的弯折 | 保持 `method = "pchip"`，PCHIP 保形特性已避免此问题 |
| 较长缺失段（如 0.3 秒）被保留为 NaN | 增大 `max_gap_frames`（如 25 → 30） |
| 插值结果过于线性、不自然 | 确保 `min_valid_points ≥ 4`，使三次插值生效 |

### 滤波（`user_params.py` postprocess.filter）

| 现象 | 调节方向 |
|---|---|
| 滤波后仍有明显高频毛刺 | 降低 `cutoff_hz`（如 6.0 → 4.0），或提高 `butterworth_order`（如 4 → 6） |
| 滤波后运动幅度明显被压缩（过度滤波） | 提高 `cutoff_hz`（如 6.0 → 8.0），或换用 `"savgol"` 方法 |
| 滤波后出现相位偏移（峰值位置变化） | 已使用零相位滤波（`sosfiltfilt`），理论上无相位偏移；若仍有问题，检查数据连续性 |
