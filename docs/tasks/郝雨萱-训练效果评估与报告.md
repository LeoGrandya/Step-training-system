# 郝雨萱：训练效果评估与历史报告

> 模块：乒乓球步伐训练效果评估系统  
> 依赖：许婉其 `kinematics_datasets` 与现有 report payload

## 目标

基于运动学摘要生成训练效果评估，并兼容历史报告与对比接口。

## 必读

- [数据库架构与开发约定](../数据库架构与开发约定.md)
- `web_1/backend/models.py` → `EvaluationRecord`, `Report`
- `web_1/backend/repositories.py` → 现有 reports 相关函数
- `web_1/backend/analysis/report_ui_builder.py`（了解 payload 形状）

## 负责表

| 表 | 说明 |
|----|------|
| `evaluation_records` | 评分、等级、建议 JSON |
| `reports` | 轻量摘要，兼容 `/api/reports` |

## 负责接口

| 类型 | 路径 |
|------|------|
| 兼容 | `GET/POST /api/reports`、`GET/DELETE /api/reports/<id>`、`GET /api/reports/compare` |
| 建议新增 | `/api/v1/evaluations`、`/api/v1/evaluations/<id>` |

查询参数：`subject_id`、`score`、`grade`、`mode`、`start_date`、`end_date`、`limit`、`offset`。

## 前端

- 训练效果评估页
- 历史报告页
- 报告对比页

## 实现要点

1. **数据源**：只读 `kinematics_datasets` / 已有 report JSON，**不重新跑视频分析**。
2. **兼容**：`/api/reports` 返回字段与旧前端一致（camelCase 等见 `repositories` 序列化）。
3. **关联**：`evaluation_records` 填 `analysis_job_id`、`kinematics_dataset_id`、`subject_id`。
4. **唯一**：`reports` 上 `subject_id` + `job_id` 唯一约束。

## 边界（不要碰）

- 新建 `analysis_results` 或第二套运动学表
- 改 pipeline 算法或 `analysis_jobs` 状态机（协调许婉其）

## 验收标准

- [ ] 分析完成后可生成 `evaluation_records` 与 `reports`
- [ ] 报告 CRUD、搜索、分页、多报告对比
- [ ] 评估可关联受试者、任务、运动学数据集
- [ ] `cd frontend; npm.cmd test; npm.cmd run build` 通过

## 联调依赖

- **等待**：至少一条完整 `kinematics_datasets` 记录
- **最后**接入全链路验收
