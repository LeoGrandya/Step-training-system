# 金彦廷：训练配置、视频与权限

> 模块：乒乓球步伐训练数据管理系统  
> 依赖：陈彦竹 `subjects`；雷润华 `route_definitions`

## 目标

训练参数配置、左右机位视频元数据、最小 RBAC（管理员 / 成员 / 只读）。

## 必读

- [数据库架构与开发约定](../数据库架构与开发约定.md) — 文件不进库原则
- `web_1/backend/models.py` → `TrainingConfig`, `TrainingVideo`, `Account`, `Role`, `Permission` 及关联表
- `v1.py` → `POST /api/analysis/jobs` 上传逻辑（需写入 `training_videos`）

## 负责表

| 表 | 说明 |
|----|------|
| `training_configs` | 训练参数、模式、分析档位快照 |
| `training_videos` | 仅路径与元数据 |
| `accounts`, `roles`, `permissions`, `account_roles`, `role_permissions` | 最小 RBAC |

## 负责接口

| 类型 | 路径 |
|------|------|
| 建议新增 | `/api/v1/training-configs`、`/api/v1/training-videos` |
| 建议新增 | `/api/v1/accounts`、`/api/v1/roles`（含权限绑定） |
| 兼容改造 | `POST /api/analysis/jobs` 上传后写入 `training_videos` |

查询参数：`subject_id`、`route_definition_id`、`status`、`limit`、`offset`。

## 前端

- 训练参数配置页
- 训练视频管理页
- 权限管理页
- 上传页的训练配置选择区

## 实现要点

1. **视频**：文件存磁盘；MySQL 只存 `left_video_path`、`right_video_path`、原名、大小、`status`。
2. **权限**：后端每个写接口校验角色；前端隐藏按钮不算安全。
3. **外键**：`subject_id`、`route_definition_id` 必须存在且未软删。
4. **不改动**：`pose3d`、`video_sync`、`kinematics` 算法目录。

## 边界（不要碰）

- `analysis_jobs` 状态机细节（许婉其维护，你可写 `training_video_id` 关联）
- `kinematics_datasets`、`evaluation_records`（许婉其、郝雨萱）
