# 许婉其：分析任务与运动学数据索引

> 模块：乒乓球步伐训练关节运动学数据管理系统  
> 依赖：金彦廷 `training_videos`；现有 `backend/analysis/` pipeline

## 目标

把现有分析任务与运动学产物接入 MySQL，形成可查询索引；**不重写算法**。

## 必读

- [数据库架构与开发约定](../数据库架构与开发约定.md) — 第 7、8 节
- `web_1/backend/analysis/pipeline_runner.py`
- `web_1/backend/repositories.py` → `upsert_analysis_job_from_record`
- `web_1/v1.py` → 分析相关路由

## 负责表

| 表 | 说明 |
|----|------|
| `analysis_jobs` | 任务状态主索引（与 JobStore 双写，MySQL 为准） |
| `kinematics_datasets` | CSV/JSON 路径与摘要 |
| `kinematics_frame_metrics` | 关键逐帧指标或抽样 |
| `analysis_results` | 兼容旧 result store，勿与郝雨萱重复建表 |

## 负责接口

| 类型 | 路径 |
|------|------|
| 兼容（保持行为） | `POST /api/analysis/jobs`、`GET .../jobs/<id>`、`GET .../result` |
| 建议新增 | `/api/v1/kinematics-datasets`、`/api/v1/kinematics-datasets/<id>/metrics` |

查询参数：`subject_id`、`status`、`stage`、`start_date`、`end_date`、`limit`、`offset`。

## 前端

- 分析任务状态页
- 运动学数据管理页
- 关节运动学详情页

## 实现要点

1. **Pipeline 完成钩子**：写入 `kinematics_datasets` 四份 CSV 路径 + `summary_json`。
2. **不全量导入 CSV**：仅抽样或聚合写入 `kinematics_frame_metrics`。
3. **JobStore**：`meta.json` 保留作调试；业务列表以 MySQL 查询为主。
4. **状态同步**：沿用 `_write_job_status_to_mysql`，补全 `stage`、`error`、`completed_at` 等字段。

## 边界（不要碰）

- 新建 `evaluation_records` 逻辑（郝雨萱）
- 修改 pose3d / video_sync / kinematics 核心算法
- 重复造「评估结果表」
