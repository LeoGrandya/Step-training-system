# 陈彦竹：受试者与基础步伐

> 模块：乒乓球步伐及受试者信息管理系统  
> 你是**第一个**业务开发者：后续所有人的下拉框和外键都依赖你。

## 目标

完成受试者与乒乓球基本步伐的基础管理，为训练、分析、报告提供稳定主数据。

## 必读

- [数据库架构与开发约定](../数据库架构与开发约定.md) — 第 3、5、6 节
- `web_1/backend/models.py` → `Subject`, `FootworkType`
- `web_1/backend/repositories.py` → 现有 `/api/users` 实现（在此基础上扩展）

## 负责表

| 表 | 说明 |
|----|------|
| `subjects` | 受试者；**不是**登录账号 |
| `footwork_types` | 基础步伐：跨步、并步、撤步、两点、三点、四点等 |

## 负责接口

| 类型 | 路径 |
|------|------|
| 兼容（必须保持） | `GET/POST /api/users`、`GET/PUT/DELETE /api/users/<id>` |
| 建议新增 | `GET/POST /api/v1/subjects`、`GET/PUT/DELETE /api/v1/subjects/<id>` |
| 建议新增 | `GET/POST /api/v1/footwork-types`、`GET/PUT/DELETE /api/v1/footwork-types/<id>` |

列表查询参数：`keyword`、`limit`、`offset`、`is_active`。

## 前端

- 受试者管理页
- 基础步伐管理页
- 全站受试者 / 步伐下拉框（与雷润华、金彦廷页面联调）

## 实现要点

1. **分页**：新接口返回 `items`、`total`、`limit`、`offset`。
2. **软删除**：`is_active=false` 或 `deleted_at`；已被 `training_configs`、`training_videos`、`analysis_jobs` 引用的记录禁止物理删除。
3. **校验**：姓名必填、数值字段类型、步伐 `code` 唯一。
4. **搜索**：`keyword` 匹配 `name`（步伐可同时搜 `code`）。

## 边界（不要碰）

- `accounts` / `roles` / 权限（金彦廷）
- `/api/analysis/jobs`、`/api/reports`（许婉其、郝雨萱）
- 不要把 `subjects` 当系统登录用户

## 验收标准

- [ ] 基础步伐增删改查、下拉、搜索、分页
- [ ] 删除被引用数据时历史任务/报告仍可查（软删）
