# 雷润华：训练路线与显示

> 模块：乒乓球步伐训练指导及显示系统  
> 依赖：陈彦竹的 `footwork_types`（及可选 `subjects` 展示）

## 目标

完成训练路线定义、动作要求、节奏控制与训练页显示所需的数据结构。

## 必读

- [数据库架构与开发约定](../数据库架构与开发约定.md)
- `web_1/backend/models.py` → `RouteDefinition`, `RouteStep`
- `v1.py` 中现有 `/api/custom-footworks`（兼容入口）

## 负责表

| 表 | 说明 |
|----|------|
| `route_definitions` | 训练路线方案（承接旧 custom footwork） |
| `route_steps` | 每步：宫格、顺序、动作类型、节奏 |

## 负责接口

| 类型 | 路径 |
|------|------|
| 兼容 | `GET/POST/PUT/DELETE /api/custom-footworks` |
| 建议新增 | `/api/v1/routes`、`/api/v1/routes/<id>/steps` |

查询参数：`keyword`、`footwork_type_id`、`limit`、`offset`。

## 前端

- 训练路线定义页
- 训练路线显示页
- 节奏调控
- 当前训练页中的自定义路线选择区

## 实现要点

1. **不要新建**第二套 `custom_footworks` 主表；旧接口读写 `route_definitions`。
2. **步骤**：创建/更新路线时同步维护 `route_steps`（`step_order` 唯一）。
3. **唯一约束**：`active_name_sequence_hash VARCHAR(64) NULL UNIQUE`；活跃路线写入 SHA-256 hash，软删除置 `NULL` 后允许重建同名同路线，重复活跃路线返回 409。
4. **节奏**：`rhythm_json` 与步骤级 `dwell_ms` / `rhythm_ms` 与前端显示一致。
5. **删除**：`route_steps` 可 CASCADE；已被训练记录引用的路线仅软删。

## Schema 修复补充（2026-06-04）

- 路线重复判断不要再依赖 `name_norm + sequence_canon` 的长字段复合唯一索引；实际唯一键是 `active_name_sequence_hash VARCHAR(64) NULL UNIQUE`。
- 创建或更新路线时，由后端根据 `name_norm` 和 `sequence_canon` 生成 SHA-256 hash；重复活跃路线仍返回 409。
- 软删除路线时必须清空 `active_name_sequence_hash`，否则同名同路线无法重新创建。
- `rhythm_json` 在 ORM 层是 `JsonText`，数据库层是 `TEXT`；需要搜索、统计、分页的字段不要只放进 JSON。
