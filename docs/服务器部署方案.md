# pose3d 慧步乒乓 — 服务器部署方案（2026-06-11 更新）

> 目标服务器：阿里云轻量服务器（Ubuntu 24.04, 1.6GB RAM, 40GB 磁盘）
> 域名：magic-cloak.com（Cloudflare）
> 已有站点：sus-electric (Node.js, port 4174, cloudflared tunnel)

## 本版本重大变化（2026-06-11）

- **分析报告页面完全重写**：使用 my-pingpong-project 原版组件（12 个 ECharts 组件，含 mock 数据降级），通过 `PingpongReportView.vue` 渲染
- **前端构建产物变化**：JS bundle 增大至 ~1.4MB（含完整 ECharts 5.6），CSS ~121KB（含 Tailwind 4 + 自定义工具类）

- **报告页数据管道**：`reportAdapter.js` 将后端 CSV/XLSX 数据转为 ECharts options → 注入组件 props → 无数据时自动降级为 mock 视觉效果
- **新增前端文件**：`frontend/src/utils/chartTheme.js`、12 个重构的 report 组件（`components/report/*.vue`）、`views/PingpongReportView.vue`

## 关键决策

- **SQLite 而非 MySQL**：服务器仅 1.6GB 内存，MySQL 常驻 300-400MB 会挤占分析算力。SQLite 零内存开销。
- **本地代码不动**：`db.py` 第 27-34 行已支持环境变量覆盖，服务器设 `DATABASE_URL=sqlite:///...` 即可，代码零变更。
- **不用 Docker**：轻量服务器内存紧张，直接用 venv + systemd。
- **gunicorn 2 workers**：1.6GB 内存下，2 个 worker + 分析线程池勉强够用。如果 OOM，降为 1 worker。

| 变量 | 值 | 说明 |
|------|----|------|

### 数据库（开发/调试用，服务端通常不设）
| 变量 | 默认值 | 说明 |
|------|--------|------|
| `POSE3D_DATABASE_URL` | — | 优先级高于 DATABASE_URL |

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `ANALYSIS_MAX_WORKERS` | `1` | 分析线程池大小。1.6GB 内存建议 1（串行分析，避免 OOM） |
| `ANALYSIS_MAX_QUEUE` | `8` | 最大排队任务数 |

| 变量 | 默认值 | 说明 |

| `ANALYSIS_KEEP_INTERMEDIATES` | `0` (删除) | 设为 `1` 保留 pose3d 中间产物 |

### 模块路径
| 变量 | 默认值 | 说明 |
|------|--------|------|
| `POSE3D_PROJECT_ROOT` | Flask 自动推断 | pose3d 模块根目录。部署在 `/opt/pose3d` 时通常无需设置 |

### 其他
| 变量 | 默认值 | 说明 |

| `FLASK_USE_RELOADER` | `0` | 生产环境应保持关闭 |

## 部署步骤

### 1. 安装系统依赖

> **Python 版本要求**：项目开发使用 Python 3.14，但服务器部署建议 Python 3.12（Ubuntu 24.04 官方源，MediaPipe 兼容性最佳）。`footwork_kinematics_universal2` 和 `pose3d_project_2.0_fixed` 中部分 numpy/scipy 调用在 3.14 上可能有兼容性警告，3.12 最稳定。

```bash
apt update
apt install ffmpeg libgl1 python3.12-venv git rsync sqlite3
```

> **ffmpeg**：视频读写。**libgl1**：MediaPipe OpenGL。**rsync**：前端文件同步。**sqlite3**：数据库备份。

### 2. 克隆项目

```bash
git clone <repo-url> /opt/pose3d
cd /opt/pose3d
```

### 3. 创建虚拟环境并安装 Python 依赖

```bash
python3.12 -m venv .venv
.venv/bin/pip install pip --upgrade
.venv/bin/pip install -r web_1/requirements.txt
# gunicorn 未在 requirements.txt 中（该文件是开发环境最小集），生产部署需要单独安装
.venv/bin/pip install gunicorn>=23
```

### 4. 构建前端

**在本地开发机执行**（服务器 1.6GB 内存无法运行 `npm run build`）：

```powershell
cd frontend
npm run build
```

构建产物在 `web_1/static/spa/`。首次部署创建目录后上传，后续更新用 rsync（自动清理旧文件）：

```powershell
# 首次
ssh cloak "mkdir -p /opt/pose3d/web_1/static/spa"
scp -r web_1/static/spa/* cloak:/opt/pose3d/web_1/static/spa/

# 后续更新（推荐 rsync，自动清理旧 hash 文件）
rsync -avz --delete web_1/static/spa/ cloak:/opt/pose3d/web_1/static/spa/
```

> **注意**：不是 `frontend/dist/*`。Vite 配置的 `outDir` 直接指向 `../web_1/static/spa/`。

### 5. 初始化数据库

```bash
cd /opt/pose3d/web_1
mkdir -p data
/opt/pose3d/.venv/bin/python -c "
from backend.db import init_database
init_database()
print('DB ready')
"
```

首次部署时会自动创建 `data/app.db` 及所有表。

### 6. 创建 systemd 服务

创建专用用户（安全最佳实践）：

```bash
useradd -r -s /usr/sbin/nologin pose3d
chown -R pose3d:pose3d /opt/pose3d
```

文件：`/etc/systemd/system/pose3d.service`

```ini
[Unit]
Description=pose3d 慧步乒乓 双机位分析系统
After=network.target

[Service]
Type=simple
User=pose3d
WorkingDirectory=/opt/pose3d/web_1
Environment="DATABASE_URL=sqlite:////opt/pose3d/web_1/data/app.db"
Environment="ANALYSIS_MAX_WORKERS=1"
Environment="ANALYSIS_KEEP_SYNC_VIDEOS=0"
Environment="ANALYSIS_KEEP_INTERMEDIATES=0"
ExecStart=/opt/pose3d/.venv/bin/gunicorn -w 2 -b 127.0.0.1:5000 --timeout 600 v1:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

> **`--timeout 600`**：分析任务可能耗时 5-10 分钟，默认 30s 超时不够。
> **`User=pose3d`**：非 root 运行，降低安全风险。分析产物目录 `web_1/jobs/` 需确保该用户可写。

启动：

```bash
systemctl daemon-reload
systemctl enable --now pose3d
```

### 7. 添加 cloudflared ingress

编辑 `/opt/cloudflared/config.yml`，在 `ingress` 段加：

```yaml
  - hostname: analysis.magic-cloak.com
    service: http://127.0.0.1:5000
```

重启：

```bash
systemctl restart cloudflared
```

### 8. Cloudflare DNS 添加 CNAME 记录

```
类型: CNAME
名称: analysis
目标: magic-cloak.com
代理: 开启（橙云）
```

## 日常更新流程

### 仅前端更新（最常见）

```powershell
# 本地
cd frontend; npm run build
# 使用 rsync 自动清理旧 hash 文件
rsync -avz --delete web_1/static/spa/ cloak:/opt/pose3d/web_1/static/spa/
```

### 后端 + 前端更新

```powershell
# 本地
cd frontend; npm run build
git push

# 服务器：同步前端 + 拉取代码 + 重启
rsync -avz --delete web_1/static/spa/ cloak:/opt/pose3d/web_1/static/spa/
ssh cloak "cd /opt/pose3d && git pull && systemctl restart pose3d"
# 如果有 Python 依赖变更，加：.venv/bin/pip install -r web_1/requirements.txt
```

### 仅后端更新（无前端变更）

```bash
ssh cloak "cd /opt/pose3d && git pull && systemctl restart pose3d"
```

## 资源预期

| 模式 | CPU | 峰值内存 | 预计耗时（30s 双机位视频） |
|------|-----|----------|--------------------------|
| 日常空闲 | ~2% | ~180MB（gunicorn 2 workers + Flask） | - |
| fast | 2 核满载 | ~550MB | 2-4 分钟 |
| balanced | 2 核满载 | ~750MB | 4-6 分钟 |
| quality | 2 核满载 | ~950MB | 6-10 分钟 |

> **OOM 风险**：balanced/quality 模式下，分析内存 + gunicorn worker 内存 + 系统内存可能接近 1.6GB 上限。如果频繁 OOM：
> 1. 降 gunicorn workers 为 1（改 systemd 的 `-w 1`）
> 2. 设 `ANALYSIS_MAX_WORKERS=1`（已在上方配置中设置）
> 3. 添加 swap：`fallocate -l 1G /swapfile && chmod 600 /swapfile && mkswap /swapfile && swapon /swapfile`

## 磁盘管理

### 空间占用来源

| 来源 | 路径 | 清理策略 |
|------|------|---------|
| 分析任务产物 | `web_1/jobs/` | 可安全删除整个目录（建议保留最近 10 个任务） |
| 数据库 | `web_1/data/app.db` | **不可删除**（业务数据） |
| 上传视频 | `web_1/jobs/*/input/` | 设 `ANALYSIS_KEEP_INPUT_VIDEOS=0` 自动删除 |
| Python 缓存 | `__pycache__/` | `find /opt/pose3d -type d -name __pycache__ -exec rm -rf {} +` |

### 建议 crontab

```bash
# 每天凌晨 3 点清理超过 7 天的分析任务目录
0 3 * * * find /opt/pose3d/web_1/jobs -maxdepth 1 -type d -mtime +7 -exec rm -rf {} \;
```

## 备份策略

### SQLite 数据库

```bash
# 每天凌晨 2 点备份（使用 sqlite3 .backup 确保事务安全）
0 2 * * * sqlite3 /opt/pose3d/web_1/data/app.db ".backup '/opt/backups/app-$(date +\%Y\%m\%d).db'"
```

> 优先用 `sqlite3 .backup` 而非 `cp`：`.backup` 在备份期间持有共享锁，不会阻塞读操作。40GB 磁盘建议保留最近 7 天备份。

依赖 git，无需额外备份。

## 故障排查

### 前端页面白屏
1. 浏览器 F12 → Console 看有无 JS 报错
2. 确认 `web_1/static/spa/` 下有 `index.html` 和 `assets/` 目录
3. 确认 `site/` 目录完整上传（legacy HTML + assets）
4. 清除浏览器缓存（Ctrl+F5）

### 报告页无数据
1. 确认 job 状态是 `done`：检查 `web_1/jobs/<job_id>/meta.json`
2. 确认 `report_payload.json` 存在：`web_1/jobs/<job_id>/report/report_payload.json`
3. 确认受试者信息已写入 job meta：检查 meta.json 中 `user_id` 字段

### 分析任务失败
1. `journalctl -u pose3d -n 50` 查看错误日志
2. 确认 ffmpeg 可用：`ffmpeg -version`
3. 确认磁盘空间：`df -h`
4. 检查 OOM：`dmesg | grep -i oom`

### gunicorn 启动失败
1. `cd /opt/pose3d/web_1 && /opt/pose3d/.venv/bin/python v1.py` 直接运行查看错误
2. 检查 Python 依赖：`/opt/pose3d/.venv/bin/pip check`
3. 确认 `POSE3D_PROJECT_ROOT` 未设置或指向正确路径
