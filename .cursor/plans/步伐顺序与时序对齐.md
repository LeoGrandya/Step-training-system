# 步伐顺序与时序对齐（修订版）

## 相对上一版的变更（用户迭代）

**小垫步（原“不单独亮灯”方案已作废）**

- **不再**按文档“不单独亮灯”处理。
- **从预置步伐下拉中移除** `pad-step`，避免与“仅自由练习使用”混淆。
- **放入自由练习场景的自定义步伐库**：以一条（或多条）可载入的自定义模板形式存在，用户从「已保存的自定义步伐」载入。
- **保留亮灯**：训练时仍走九宫格 `active/return` 高亮逻辑。
- **亮灯速度由慢到快**：同一套小垫步序列在时间上应随进度加速（见下文实现选项）。

其余对齐目标不变：严格按文档修正单步、撤步、全台摆速、推侧扑回反手等顺序；时间参数按文档区间**中位值**映射；全台摆速收敛为 `5→4→5` / `5→6→5`；推侧扑回反手末尾回 `5`。

## 小垫步「由慢到快」实现要点（需在开发时选定其一）

当前引擎对每一步使用全局 `T_on` + `T_cycle`（或随机区间），**不自带“同序列内渐变加速”**。要实现由慢到快，需在下列方案中落地一种（推荐 A 或 B）：

| 方案 | 做法 | 优点 | 代价 |
|------|------|------|------|
| **A. 模板元数据** | 自定义库条目增加可选字段 `accel: { fromMs, toMs, steps }`，仅自由练习载入该模板时，`executeNextStep` 按步索引计算当前步的 `effectiveCycleMs` | 语义清晰、可复用到其他模板 | 需扩展库模型与前端执行路径 |
| **B. 语法扩展** | 在自定义序列中支持可选片段，例如 `T0.25` 表示下一步起使用周期 0.25s（或 `C0.22` 仅改 gap） | 不依赖库字段扩展 | 需扩展 `parseCustomSequence` 与文档说明 |
| **C. 离散近似** | 用较长 `W` + 固定较快 `T_cycle` 分段手搓多段序列（多段重复） | 改动最小 | 难精确、维护差 |

**推荐**：优先 **A**（与「仅在自由练习从库载入」一致，不把复杂度塞进通用语法）。

## 验收清单（小垫步专项）

- 预置 `stepType` 中**无**小垫步项。
- 自由练习下可从自定义库载入「小垫步」，训练时**有亮灯**。
- 同一次训练中，后半段灯切换间隔**明显短于**前半段（或由慢到快的单调趋势）。
- 练习评估 / 能力测试：不因移除预置小垫步而报错；能力测试固定序列不受影响。

## 最终推荐方案（最佳解决方案 · 整合论述）

本节把前文结论与修订版计划收束为**一套可执行、可验收**的落地方案（文档真源 + 预置课表 + 小垫步独立形态 + 变速实现选型）。

### 1. 总体原则（三条）

1. **文档是唯一真源**：步伐顺序与时间区间以 `乒乓球九宫格训练亮灯时间间隔及参考文献_最终修正版.txt` 为准；时间区间在系统中落地为**固定中位值**（秒），避免每步随机带来的不可复现性。
2. **预置步伐 = 标准课表**：类型有限、顺序固定、节奏可一键套用文档默认值；**不把「小垫步」放在预置里**。
3. **小垫步 = 独立产品形态**：仅在**自由练习**场景，通过**自定义步伐库**提供官方种子模板；**必须亮灯**；**同一次训练内周期单调变快**（由慢到快）。

### 2. 架构分层（推荐）

| 层 | 职责 |
|----|------|
| **序列层** | 预置 `stepLibrary` 按文档修正顺序；预置下拉**移除** `pad-step`。 |
| **节奏层** | `stepType → { tOn, tCycle }`（及测试模式下的 min/max 策略）由文档中位值表驱动；切换步伐时**自动回填**设置项，用户仍可手动覆盖。 |
| **小垫步专用** | 不强行复用「全程同一 T_cycle」假设：对绑定**模板级加速 profile** 的库条目，在 `executeNextStep` 中按步索引计算 `effectiveCycleMs`（及可选 `effectiveTOnMs`）。 |

### 3. 小垫步「由慢到快」的选定结论

- **采用方案 A：模板元数据**（见上文方案表）。理由：语义清晰、与「仅库载入」一致、不污染 `parseCustomSequence` 通用语法、后续其他变速课可复用。
- 字段形态示例（实现时可微调命名）：`timingProfile: { kind: "linear_accel", tOnStart, tOnEnd, cycleStart, cycleEnd }`，或与文档区间对齐的数值；约束每步 `cycle_i >= tOn_i`。

### 4. 与文档对齐的落地清单（顺序 + 时间）

**顺序（预置内）**

- `single-step`：目标格集合与文档一致（去除文档未列的侧台格若适用）。
- `back-step`：改为文档路径 `5 ↔ 7/8/9`（替换当前固定单侧台逻辑）。
- `full-table`：收敛为 **`5→4→5` / `5→6→5`**（持拍手或左右镜像策略与现有一致）。
- `push-side-pounce-back`：补 **末尾回 5**，与文档 `…→5` 一致。
- `push-side-pounce`：主序列与文档一致；若内置长停顿与文档冲突，改为**可配置**或按文档节奏重算。

**时间**

- 维护 **文档区间 → 中位值** 映射表；各 `stepType` 切换时写入 `T_on` / `T_cycle`（及测试随机范围若需要）。
- 页面默认值与文案与「文档对齐默认值」一致；可选增加「一键恢复文档推荐」以降低支持成本。

**小垫步**

- 预置移除 + 自由练习路径 **seed** 官方「小垫步」模板进库 + UI 引导；训练走亮灯 + **linear_accel** profile。

### 5. 整体验收标准（可打勾）

- 预置中**无**小垫步；自由练习可从库载入官方小垫步且**有灯**。
- 小垫步单轮内**后半段明显快于前半段**（周期单调下降或可量化曲线）。
- 其余已在对比表中标注的步伐**顺序**与文档一致。
- 切换预置 `stepType` 时，时间参数自动变为该类型文档中位默认；用户手动修改后，仅在**再次切换步伐**或**点击恢复推荐**时被覆盖（避免训练中静默改参）。

### 6. 为何视为「最佳」

- **一致性**：顺序与时间双对齐，而非只改其一。
- **产品边界清晰**：小垫步不进预置、放自由练习库，符合专项节奏训练定位。
- **可维护性**：变速用模板元数据，优于手搓 `W` 链或过早扩展通用序列语法。
- **风险可控**：核心改动集中在 [`web_1/static/js/app-core.js`](web_1/static/js/app-core.js) 与 [`web_1/static/js/app-flow.js`](web_1/static/js/app-flow.js)；后端 [`web_1/v1.py`](web_1/v1.py) 可按阶段扩展 `custom_footworks` 存 profile，或首版仅前端 seed + 本地缓存。

### 7. 文档与代码侧同步

- 实施完成后更新 [`九宫格亮灯间隔_文档与项目实际对比.md`](../../九宫格亮灯间隔_文档与项目实际对比.md)：小垫步行改为「预置移除 + 自由练习库 + 亮灯 + 变速」；顺序/时序矩阵与结论同步为已实现状态。
